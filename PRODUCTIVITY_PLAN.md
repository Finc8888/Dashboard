# 🚀 План продуктивности: PHP → Golang + Спорт + Языки

> **Контекст:** PHP/JS/SQL разработчик, знакомый с DDD и юнит-тестами. Цель — постепенный переход на Go, борьба с прокрастинацией, изучение языков через Duolingo, бег по утрам.

---

## 🧠 Диагностика проблемы

Прежде чем менять распорядок — важно понять **корень** низкой продуктивности:

| Симптом | Вероятная причина |
|---|---|
| 1–2 часа фокуса в день | Когнитивное истощение, нет системы |
| Прокрастинация после | Задачи слишком большие, нет "побед" |
| Накопившийся беклог | Замкнутый круг — стресс → прокрастинация |
| Хочу учить Go, но "нет времени" | Нет выделенного слота + завышенные ожидания |

**Главный принцип:** Не добавляй нагрузку — **перераспредели** существующую энергию.

## ⚙️ Система борьбы с прокрастинацией

### Правило "2-минутного старта"
Если не хочется начинать — скажи себе: *"Я просто открою файл и напишу одну строку."*
Начало — это 80% победы.

### Метод Pomodoro (адаптированный)
```
25 мин работы → 5 мин отдыха → повтор
После 4 помидоров → 20 мин длинный перерыв
```
Используй: [Pomofocus.io](https://pomofocus.io) или расширение для браузера.

### "Следующее действие" (GTD-подход)
Вместо задачи *"Сделать фичу X"* пиши:
→ *"Открыть файл UserService.php и добавить метод getById()"*

Мозг прокрастинирует от размытости, а не от сложности.

### Правило завершения дня
Каждый вечер записывай **3 выполненных дела** — это формирует ощущение прогресса и снижает стресс.

## 📚 План изучения Golang

### Принцип: "Go через работу"
Не учи Go в вакууме — **переписывай рабочие утилиты на Go**. Это даст:
- Реальный контекст (не абстрактные туториалы)
- Пользу для работы
- Быстрое освоение идиом языка

### Этапы (по 4–6 недель)

#### 🟦 Этап 1: Фундамент (Недели 1–4)
**Слот:** 45 мин утром (10:45–11:30) × 5 дней в неделю

| Неделя | Тема | Ресурс |
|---|---|---|
| 1 | Синтаксис, типы, функции | [Tour of Go](https://go.dev/tour) |
| 2 | Структуры, интерфейсы, методы | Tour of Go + практика |
| 3 | Горутины, каналы, concurrency | [Go by Example](https://gobyexample.com) |
| 4 | Работа с пакетами, модули, go mod | Документация + свой проект |

**Практика:** Ежедневно — один маленький скрипт на Go.

#### 🟩 Этап 2: Go для бэкенда (Недели 5–10)

| Тема | Параллель из PHP |
|---|---|
| `net/http` → HTTP-сервер | Laravel/Symfony роутинг |
| `database/sql` → работа с БД | Eloquent / PDO |
| Структурирование пакетов | DDD layers в PHP |
| Юнит-тесты (`testing` package) | PHPUnit |
| Middleware, Dependency Injection | Уже знаешь принципы |

**Проект этапа:** Простой REST API (аналог одного из рабочих модулей на PHP).

#### 🟨 Этап 3: Переход (Недели 11–16)

- Напиши один реальный микросервис на Go
- Настрой CI/CD pipeline для Go
- Изучи популярные фреймворки: **Gin**, **Echo**, **Chi**
- Сравни с PHP: как решать те же DDD-паттерны на Go

## 🔄 Управление рабочим беклогом

### Трёхуровневый список задач

```
🔴 КРИТИЧНО (сегодня)     — 1–2 задачи максимум
🟡 ВАЖНО (эта неделя)     — 3–5 задач
🟢 МОЖНО ПОДОЖДАТЬ        — остальное
```

**Правило:** Каждое утро выбирай **одну главную задачу дня** и выполни её до обеда.

### Еженедельный ретроспективный обзор (15 мин, пятница вечером)
1. Что сделано?
2. Что заблокировало?
3. Что изменить на следующей неделе?
4. Что добавить в Go-обучение?

## 🏃 Бег + восстановление

Вечерний бег (19:00–20:00) — отличный способ сбросить накопившееся напряжение после рабочего дня и "перезагрузить" голову перед Duolingo.

**Рекомендации:**
- Не ешь плотно за 1.5–2 часа до бега — лёгкий перекус в 18:30 идеально
- После бега — душ и сразу Duolingo: мозг после физнагрузки лучше усваивает язык
- Лёгкий/средний темп: бежишь и можешь говорить предложениями — идеальная зона
- Не добавляй интенсивность резко — это мешает сну и завтрашней продуктивности

## 📱 Duolingo (20:00–21:00)

Слот после бега — очень удачный: мозг после физнагрузки лучше запоминает новый материал.

- Включи **streak freeze** про запас
- Используй режим **Hard mode** для реальных навыков
- После 21:00 — переходи в офлайн-режим (книга, не экраны)

## 📵 Борьба с YouTube и лентой новостей

Это одна из главных причин потери продуктивных часов. YouTube и новости — это **не отдых**, это псевдо-отдых: мозг не восстанавливается, но устаёт от потока информации.

### Техника "окна дозволенного"
Не запрещай себе YouTube полностью — это вызывает обратный эффект. Вместо этого **огради рабочее время технически:**

```
Разрешённое время для YouTube/новостей:
  ✅ 16:00 – 18:30 (после рабочих блоков)
  ❌ Всё остальное время — заблокировано
```

### Инструменты блокировки (выбери один)

| Инструмент | Платформа | Что делает |
|---|---|---|
| **Cold Turkey** | Mac / Win | Жёсткая блокировка сайтов по расписанию |
| **Freedom** | Mac / Win / iOS | Синхронизирует блокировку на всех устройствах |
| **LeechBlock** | Firefox | Бесплатное расширение, блокировка по времени |
| **uBlacklist** + расписание | Chrome | Убирает сайты из результатов поиска |

> **Совет:** Установи блокировку сегодня вечером, пока есть мотивация. Завтра утром ты скажешь себе спасибо.

### Замена триггеров

Когда рука тянется к YouTube — это сигнал **усталости или скуки**, не потребности в видео.

| Вместо YouTube/новостей | Когда |
|---|---|
| 5 мин прогулка от стола | В рабочее время |
| Стакан воды + 10 глубоких вдохов | При импульсе отвлечься |
| Открыть go.dev/tour на следующий урок | Когда скучно между задачами |
| Записать мысль в дневник | Если что-то тревожит |

### Утро без телефона
Первые 30 минут после подъёма — **без телефона и новостей**. Это одна из самых мощных привычек для продуктивности: ты сам задаёшь тон дню, а не алгоритм ленты.

## ⚠️ Главные ловушки — избегай

| Ловушка | Решение |
|---|---|
| "Начну изучать Go с понедельника" | Начни сегодня с 1 строки кода |
| Учить Go по 3+ часа в выходные | Лучше 30 мин каждый день |
| Читать статьи про Go вместо написания кода | 80% практика, 20% теория |
| "Только одно видео на YouTube" | Блокировщик решает вопрос за тебя |
| Новости утром → тревога → прокрастинация | Новости строго после 16:00 |
| Пропустить бег → чувствуешь вину → прокрастинация | Пропуск = норма, не катастрофа |
| Книга затянула, лёг в 01:00 | Будильник-напоминание в 22:20 "закладка" |
