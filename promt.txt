Лимиты сняты продолжи  выполнение предыдущей задачи только я немного хочу изменить логику. Я откатил измеенения в @www/data/dashboard-data-default.json этот файл должен быть единым для импорта и экспорта поэтому ключи тоже должны быть однинаковыми. Только убери экранирование из таких файлов а преобразование в строку для хранения в local storage делай в коде. Принцип дефолтных настроек виджетов должен быть следующим. Если пользователь только зарегистрировался и первый раз заходит в Dashboard UI. То он должен автоматически просто импортировать дефолтный файл экспорта @www/data/dashboard-data-default.json для этого уже есть готовые функции в коде. Только поведение которое было захардкожено например отображение одного лишь todo list только зарегистрированному пользователю - должно браться из @www/data/dashboard-data-default.json. Отредактируй этот файл добавив для ключа prod_widgets_gladys.visible - true только для todo лист и сдеай его пустым. Так же остальные данные которые не относятся к виджетам а относятся к конкретному пользователю например gladys_user или gladys_token - сделай  дефолтном файле экспорта пустыми и вычисляй их програмно. Как только пользователь сделал свой первый экспорт настроект, то эти поля станут заполнеными и при импорте такие данные будут браться оттуда. Так же как только админ даст права на виджеты пользователю - то пользователь уже сам сможет сделать другие виджеты активными из меню а так же изменить их порядок. Тут главное учесть что после активации доступа админом конкретному пользователю всех виджетов. Пользователь уже может внести изменения в виджеты которые были ему доступны по умолчанию например туду лист. Таким образом при отображении всех виджетов должен быть гладкий merge между измененными пользователем данными и дефолтными.
