# AI Ассистент для Dashboard — Мозговой штурм

> Дата: 2026-04-03
> Контекст: gemma3:4b на i7-1360P / 16GB RAM / Intel Iris Xe — слишком медленно (~5 мин на ответ), качество низкое

---

## Проблемы текущего решения

| Проблема | Причина |
|----------|---------|
| Ответ ~5 минут | CPU-only inference, i7-1360P без дискретной GPU |
| Неточные ответы | 4B модель плохо понимает структурированный контекст |
| Путает стикеры с задачами | Модель не различает семантику разных секций данных |
| num_ctx 16K на CPU = ещё медленнее | Больше контекст = больше вычислений на каждый токен |

## Вариант 1: Выделенное железо с GPU (рекомендуемый для локальности)

### 1A. Мини-ПК с NVIDIA GPU

**Конфигурация:**
- **NVIDIA Jetson Orin Nano Super** (~$250)
  - 8GB unified RAM, 1024 CUDA cores
  - Потянет: gemma3:4b (быстро), gemma3:12b (приемлемо)
  - Размер: как Raspberry Pi
  - Энергопотребление: 7-25W

- **Мини-ПК + eGPU (внешняя видеокарта)**
  - Intel NUC / Beelink + Thunderbolt eGPU enclosure + RTX 3060 12GB
  - ~$400-600 за всё
  - Потянет: 12B-27B модели с хорошей скоростью
  - Минус: громоздко, шум от GPU

**Скорость ожидания:**
| Модель | Jetson Orin Nano | RTX 3060 12GB | RTX 4060 8GB |
|--------|-----------------|---------------|--------------|
| gemma3:4b | ~15 tok/s | ~40 tok/s | ~50 tok/s |
| gemma3:12b | ~5 tok/s | ~20 tok/s | ~25 tok/s |
| gemma3:27b | не влезет | ~8 tok/s | не влезет (VRAM) |

### 1B. Самосборный тихий сервер

**Конфигурация:**
- Корпус: SilverStone SG13 / Node 202 (mini-ITX)
- CPU: любой i3/i5 (только для системы)
- RAM: 16-32 GB DDR4
- GPU: **RTX 3060 12GB** (~$200 б/у) — лучший price/VRAM
- SSD: 256GB (система + модели)
- БП: 450W SFX
- **Итого: ~$400-500 из б/у комплектующих**

**Плюсы:** тихий, мощный, расширяемый, хороший VRAM
**Минусы:** нужно собирать, потребление ~100-200W под нагрузкой

### 1C. Raspberry Pi 5 — НЕ рекомендуется

- 8GB RAM, ARM CPU, нет GPU для inference
- gemma3:4b ~0.5-1 tok/s — хуже чем на ноутбуке
- Подходит для хостинга Dashboard UI, но НЕ для AI модели

### 1D. Arduino — НЕ применимо

- Микроконтроллер, нет ОС, нет RAM для LLM
- Полностью не подходит для задачи

## Вариант 2: Облачные бесплатные решения

### 2A. Google Colab Free

- **GPU:** T4 16GB (бесплатно, с ограничениями)
- Можно запустить Ollama + gemma3:12b
- **Минусы:** сессии умирают через 1-4 часа, нет постоянного URL, нужен ngrok/cloudflared для туннеля
- **Вердикт:** для экспериментов ОК, для постоянного использования — нет

### 2B. Kaggle Notebooks

- **GPU:** T4 16GB или P100, 30 часов/неделю
- Те же ограничения что и Colab
- **Вердикт:** аналогично, не для production

### 2C. Oracle Cloud Free Tier

- **ARM VM:** 4 OCPU + 24GB RAM (бесплатно навсегда)
- Можно запустить gemma3:4b-12b на CPU
- ARM Ampere быстрее x86 для inference: ~3-5 tok/s для 4B
- **Плюсы:** бесплатно, постоянный IP, надёжно
- **Минусы:** всё ещё CPU, медленнее чем GPU; могут отобрать ресурсы при idle

### 2D. Hugging Face Spaces (ZeroGPU)

- Бесплатная GPU-квота для inference
- Можно развернуть через Gradio/FastAPI
- **Минусы:** холодный старт ~30-60 сек, ограничения по времени GPU

## Вариант 3: Claude API через Pro подписку

### Важно: Pro подписка ≠ бесплатный API

Pro подписка ($20/мес) даёт доступ к **claude.ai веб-интерфейсу**, но **НЕ** к API.
API тарифицируется отдельно: ~$3/M input tokens, ~$15/M output tokens (Claude Sonnet).

Однако есть варианты:

### 3A. Claude через MCP (Model Context Protocol)

Если хочешь использовать Claude из Dashboard — можно через промежуточный сервер:

```
[Dashboard Widget] → [Твой Go-сервер на VPS] → [Claude API]
```

**Стоимость:** ~$0.01-0.05 за запрос (зависит от контекста)
При 20-30 запросов в день: ~$10-30/мес

**Плюсы:** мгновенные ответы, высокое качество, понимание русского
**Минусы:** не бесплатно, не локально, данные уходят в облако

### 3B. Claude Code как CLI-ассистент (уже есть)

Вместо виджета — использовать Claude Code для анализа экспорта:
```bash
cat dashboard-export.json | claude "проанализируй мои задачи"
```
Это входит в Pro подписку (через Claude Code). Не требует API ключа.
**Минус:** не интегрировано в UI.

└─────────────────────────────────────────────────────┘

### Рекомендуемая конфигурация Home Server

**Бюджетный (~$300-400 б/у):**
- Корпус: mini-ITX (SilverStone, Fractal Node)
- CPU: Intel i5-12400 / AMD Ryzen 5 5600
- RAM: 16GB DDR4
- GPU: **NVIDIA RTX 3060 12GB** (~$180-220 б/у)
- SSD: 256GB NVMe
- БП: 450W
- Ubuntu Server 24.04

**На нём запускается:**
- Docker + docker-compose
- Caddy (Dashboard UI + Auth Gateway + все проекты)
- Ollama + gemma3:12b (быстрый inference на GPU)
- WireGuard клиент → подключение к VPS

**Средний (~$500-700):**
- То же + RTX 3090 24GB (б/у ~$400)
- Потянет gemma3:27b и модели класса Llama 3 70B Q4
- Или: 2x RTX 3060 для параллельных задач

**Компактный (~$250-350):**
- NVIDIA Jetson Orin Nano Super ($250)
- Размер как Raspberry Pi, но с GPU
- gemma3:4b-12b с достойной скоростью
- Идеально для тихого всегда-включённого сервера (7W idle)

### Сервисы на Home Server (docker-compose)

```yaml
services:
  ollama:
    image: ollama/ollama
    deploy:
      resources:
        reservations:
          devices:
            - capabilities: [gpu]
    volumes:
      - ollama_data:/root/.ollama
    ports:
      - "127.0.0.1:11434:11434"  # только локально
    restart: unless-stopped

  caddy:
    # Dashboard + Auth Gateway + проекты
    # Ollama проксируется через Caddy для auth
    ports:
      - "10.8.8.8:443:443"  # доступ только через WireGuard
    restart: unless-stopped

  wireguard:
    image: linuxserver/wireguard
    cap_add: [NET_ADMIN]
    environment:
      - WG_SERVER=your-vps-ip
    volumes:
      - ./wg0.conf:/config/wg0.conf
    restart: unless-stopped
```

### Маршрутизация запросов к Ollama через Caddy

```caddyfile
# В Caddyfile на home server
10.8.8.8:443 {
    # Dashboard AI запросы проксируются к Ollama
    handle /api/ollama/* {
        uri strip_prefix /api/ollama
        reverse_proxy localhost:11434
    }
    # Остальное — Dashboard
    handle {
        root * /srv/www
        file_server
    }
}
```

В виджете Dashboard URL меняется на:
```javascript
const AI_OLLAMA_URL = '/api/ollama';  // через Caddy reverse proxy
```

## Детальные конфигурации: Mini-ПК vs Самосборный сервер

### Что определяет производительность LLM

Единственный критичный ресурс — **VRAM (видеопамять)**. Всё остальное вторично.

| VRAM | Максимальная модель (Q4 квантизация) | Уровень качества |
|------|--------------------------------------|------------------|
| 8 GB | gemma3:12b, Llama 3.1 8B | Базовый — простые задачи |
| 12 GB | gemma3:12b комфортно, gemma4 12B | Хороший — анализ данных, саммари |
| 16 GB | gemma3:27b (тесно), Llama 3.1 70B Q2 | Хороший+ |
| 24 GB | gemma3:27b, gemma4 27B, Llama 3.1 70B Q4 | Отличный — сложный анализ, код |
| 48 GB (2x24) | Llama 3.1 70B Q8, Mixtral 8x22B | Топ — уровень GPT-4 класса |

> **gemma4** ожидается в размерах ~4B / 12B / 27B. Для 27B нужно 24GB VRAM.

## Mini-ПК vs Самосборный: итог

| Критерий | Готовый Mini-ПК + eGPU | Самосборный Mini-ITX | Самосборный ATX |
|----------|----------------------|---------------------|-----------------|
| Компактность | ★★★★★ | ★★★☆☆ | ★★☆☆☆ |
| Простота сборки | ★★★★★ (купил-включил) | ★★★☆☆ (1-2 часа) | ★★★☆☆ (1-2 часа) |
| Производительность GPU | ★★★☆☆ (потери на TB) | ★★★★★ (полная) | ★★★★★ (полная) |
| Расширяемость | ★★☆☆☆ | ★★★☆☆ (1 GPU) | ★★★★★ (2+ GPU) |
| Тишина | ★★★★☆ | ★★★☆☆ | ★★☆☆☆ |
| Цена за VRAM | ★★☆☆☆ (eGPU дорого) | ★★★★★ | ★★★★★ |
| Upgrade GPU | ★★★★★ (просто менять) | ★★★★☆ | ★★★★★ |

### Вердикт

**Для твоей задачи (Dashboard + Ollama + WireGuard) — самосборный Mini-ITX.**

Причины:
- eGPU теряет 15-25% производительности впустую
- Mini-ITX достаточно компактный для полки/шкафа
- Полноценный PCIe x16 = максимум от GPU
- Один GPU слот — это ОК, для 12B-27B моделей хватит одной карты
- Легко сменить GPU позже (3060 → 3090 → 4090)

## Анализ GPU с Ozon (апрель 2026)

### Почему RTX 3090, а не RTX 4090?

LLM inference — это задача **memory-bandwidth bound**, а не compute-bound.

| Параметр | RTX 3090 | RTX 4090 |
